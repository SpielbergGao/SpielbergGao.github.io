    项目经验

    come(社交 App)

    背景: 1. 独立开发 2.持续功能迭代
    方案: 1. 项目采用 Kotlin 语言开发 2.对使用率高的模块拆分解耦 3.对线上崩溃日志收集 4.对音 频，视频异步回收 5.控制图片绘制频率
    效果:
  • 核心模块以组合的方式将控件分离数据对视图单独进行渲染;
  • Bugly 崩溃率由 3% 降为 0.5%
  • 重构核心代码后减少5%的代码量
  • 封装MQTT并根据不同 Android 版本重写启动方式有效避免 8.0 服务奔溃问题
  • 修复主流程内存泄漏及界面卡顿有效提高用户体验
  • 定位功能封装后控制定位时机避免频繁定位引发高耗电;
  • 对 MQTT 服务二次封装及部分功能修改适配8.0服务
  • 结构采用 MVVM 分离 model 与 view 耦合
  • 音视频采用封装的管理器进行统一注册与注销 对列表进行单独数据刷新减少不必要的全局刷新引起的卡顿 ;

    1CM-OA 系统(OA 办公 App)

    背景: 1. 独立开发 2. 项目技术选型及搭建 3. 敏捷开发 4. 节省开发成本
    方案: 1. 项目采用 Kotlin 语言开发 2. 网络请求使用 Retrofit OkHttp Rxjava 3. 自定义控件使用 Builder设计模式 4. 日志回馈及修复
    效果
  • 提升 Gradle 编译速度从 5min 降低到 1min 提升开发效率;
  • 对不同核心模块架构优化，对这些 模块彻底解耦，降低维护成本约 30%;
  • 控件模块通用化提供多方支持减少 20% 开发时间;
  • 自定义弹框，对话框，日历等特殊控件; 根据不同网络环境对数据请求，图片进行不同的处理 减少电量的消耗;
  • OkHttp3 设置网络请求日志拦截，超时处理，请求失败重新提交请求，开启缓存，并对读取与 写 入进行超时设置，提高数据请求稳定性;

    儿童创客(社交 App)

    背景: 1.独立开发 2. 新功能迭代 3. 改善既有代码 4. 迭代成本高 5. 对接成本高 6. 维护既有第三方 方案: 1. 项目改用 Kotlin 重构 2. 对接方式由 Jar 替换为 aar 3. 自定义控件使用 Builder设计模式 4. 多个类型需要重复判断的对象采用抽象工厂模式重构 5. 日志回馈及修复
    效果:
    使用腾讯 UGC 完成短视频开发;
  • 代码部分由 Java 重构为 Kotlin，代码减少 50%;
  • 删除未使用的资源文件，对硬编码进行抽取 重复使用，包体积减少 10%;
  • 对常用重复的代码进行封装提高复用性，减少 15% 的代码;
  • 将 Jar 中间件包替换为 aar 减少接入错误率 10% ;
  • 使用 Unity3D 与 C# 进行 SDK 功能测试;

    儿童创客 TV(TV 盒子 App)

    背景: 1. 独立开发 2. 新功能迭代 3. 改善既有代码 4. 迭代成本高 5. 过期方法调用多 方案: 1. 项目改用 Kotlin 重构 2. 使用工厂模式重构 3. 日志回馈及修复
    效果:
  • 封装文件解压方法，网络请求，图片处理方法，减少 30% 开发时间;
  • 控制焦点事件传递增加 容错率，遥控器点击焦点获取效率提升30%;
  • 删除过期的 API 使用官方推荐上下兼容好的 API，减少50%的编译警告;
  • 根据不同分辨率和密度进行屏幕适配

    禄正支付(支付类 App)

    背景: 1. 独立开发 2. 新功能迭代 3. 改善既有代码 4. 迭代成本高
    方案: 1. 自定义控件使用 Builder设计模式 2. 多个类型需要重复判断的对象采用抽象工厂模式重构 3. 改善继承关系扩展功能改为接口实现
    效果:
  • 剔除未使用的代码与资源文件包体积减小 10MB;
  • 优化蓝牙连接提高扫描速率降低扫描范围时 间降低到 1s;
  • 对用戶的注册加入实名认证，身份证校验，手机号校验;
  • 将出现次数高代码数量多的功能模块 进行抽取，达到重复利用提升效率;
  • 支付功能采用对称加密 AES 和非对称加密 RSA 保证数据和支付安全;
  • WebView 采用腾讯 X5 提供的依赖库去除接口引起的远程代码执行漏洞域控制漏洞等添加白 名单;
